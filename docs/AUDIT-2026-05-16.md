### F-008: Stream refill failure after bytes reached GRBL did not command laser-off or latch inspection-required fault state

- **Severity:** High
- **Category:** Safety error handling / exceptional condition
- **CWE:** CWE-755 (Improper Handling of Exceptional Conditions), CWE-754 (Improper Check for Unusual or Exceptional Conditions)
- **ISO/IEC 25010:** Reliability, Functional suitability, Security
- **Risk score:** 4 x 3 x 2 = **24**
- **Location:** `src/controllers/grbl/GrblController.ts:1150`
- **Evidence:**
  ```ts
  private _handleStreamFillError(err: unknown): void {
    const message = err instanceof Error ? err.message : String(err);
    for (const cb of this._errorListeners) {
      cb(0, `GRBL stream fill failed: ${message}`);
    }
    this._abortJob();
    this._emitProgress();
  }
  ```
  Before the fix, a `gcode-stream` spool that failed while refilling the bounded send window after prior job lines had already reached the serial transport only aborted host bookkeeping.
