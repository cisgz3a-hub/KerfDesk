# Whole-Repo LightBurn Parity Audit Prompt - 2026-06-04

## Objective

Audit LaserForge 2.0 against LightBurn as the workflow reference and against GRBL as the wire-protocol authority. The goal is to identify concrete release risks, not to invent a feature wishlist.

## Repo Rules

- Work audit-only unless explicitly asked to patch production code.
- Every accepted finding needs path, module/function, trigger, failure mode, consequence, severity, confidence, and concrete fix.
- Reject stale findings that recent ADRs/tests already fixed.
- Reject broad best-practice advice without a realistic trigger path.
- Treat laser safety paths as evidence-only. A passing unit test is not enough if the behavior can mark material or leave the operator without Stop.

## Audit Lanes

1. Operator workflow: File, edit, selection, layers, preview, job setup, origin, frame, start, pause, stop, disconnect.
2. Image workflow: import raster, trace, preview, commit, convert to bitmap, image engrave, raster budgets, freeze risks.
3. Vector workflow: SVG import, fill-only geometry, transforms, text, fill hatching, layer modes, output order.
4. Output truth: preview equals emitted G-code, estimates, invariants, provenance, G-code safety.
5. Device safety: serial open/close, GRBL errors, alarms, buffer streaming, Stop reachability, frame/jog safety states.
6. Platform/process: web vs desktop parity, local-only policy, deploy/build guards, audit coverage.

## False-Positive Rejection Rules

- Do not flag trace source revalidation unless current code allows stale trace commit.
- Do not flag raster wide-gap travel if `emit-raster.ts` splits wide blank spans and tests cover it.
- Do not flag M3 fill overburn if `grbl-strategy.ts` now emits M4 for fill.
- Do not flag raw preview/order drift if the consumer routes through `prepareOutput`; only flag remaining direct compile paths.
- Do not flag lack of general cloud services unless an actual network call or dependency is present.

## Deliverables

- Human report in `audit/reports/`.
- Machine-readable findings in `audit/findings/`.
- Evidence ledger in `audit/evidence/`.
- Source coverage ledger generated by `audit/scripts/generate-source-coverage-ledger.mjs`.
- Roadmap separating P0 safety, P1 freeze/output truth, P2 LightBurn parity, and P3 process gaps.
